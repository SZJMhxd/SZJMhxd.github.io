设置MySQL

修改密码

对密码策略进行设置

在这之前 我们先要对密码策略进行设置

  1. mysql -u root -p 首先使用root用户登录

  2. 查看密码策略

  3. 修改密码策略

  4. 再次查看密码策略

修改密码

参考 MySQL修改root用户密码

使用ALTER USER修改密码

新建用户

参考 MySQL创建用户（3种方式）

  1. 使用CREATE USER新建用户

    使用 CREATE USER 创建一个用户，用户名是 test1，密码是 test1，主机名是localhost。

  2. 使用新用户登录

创建数据库

  1. 使用CREATE DATABASE创建数据库

  2. CREATE DATABASE IF NOT EXISTS test_db1
DEFAULT CHARACTER SET utf8
DEFAULT COLLATE utf8_general_ci;

  4. 使用SHOW DATABASES;查看有哪些数据库

分配数据库权限

参考mysql 创建用户、设置权限

使用GRANT命令授权用户

  • privileges：用户的操作权限，如SELECT，INSERT，UPDATE等，如果要授予所的权限则使用ALL

  • databasename：数据库名

  • tablename：表名，如果要授予该用户对所有数据库和表的相应操作权限则可用*表示，如*.*

使用Navicat远程连接

使用Navicat远程连接时遇到错误

1130- Host 16.47.201.183’is not allowed to connect to this MySQL server

这是由于Mysql配置了不支持远程连接引起的

  1. 在安装Mysql数据库的主机上登录root用户

  2. 看到当前主机配置信息

    Host列指定了允许用户登录所使用的IP，比如user=root Host=192.168.1.1。这里的意思就是说root用户只能通过192.168.1.1的客户端去访问。 user=root Host=localhost，表示只能通过本机客户端去访问。而%是个通配符，如果Host=192.168.1.%，那么就表示只要是IP地址前缀为“192.168.1.”的客户端都可以连接。如果Host=%，表示所有IP都有连接权限。

  3. 修改host

  4. 再次查看

  5. Host修改完成后记得执行flush privileges使配置立即生效

  6. 再次用navicat连接成功
